## What You'll Learn

- Create reusable prompt templates with variable placeholders
- Organise templates in a logical folder structure
- Adapt templates for different use cases
- Recognise when to create a template vs. write a one-off prompt

## The Scenario

Jamie keeps typing the same prompts over and over - meeting summaries, article analyses, file processing. Templates let you write once and reuse forever, just by filling in the blanks.

## Key Concepts

| Concept              | Description                                                         |
| -------------------- | ------------------------------------------------------------------- |
| **Template**         | A saved prompt with placeholders like [FILENAME] that you fill in   |
| **Placeholder**      | Variables in brackets that you replace with actual values           |
| **Template Library** | Organised collection of reusable prompts by category                |
| **When to Template** | Use 2-3 times = consider templating; 3+ times = definitely template |

## What You'll Create

- `templates/` folder structure with categories
- Meeting summary template with placeholders
- Project brief template
- At least one custom template for your own workflow

## Quick Reference

- **Placeholder format**: Use [CLEAR_NAME] like [FILENAME] or [ARTICLE_URL]
- **Include instructions**: Add "How to Use" section in each template
- **Stay focused**: One template = one task
- **Evolve**: Update templates as you learn what works
