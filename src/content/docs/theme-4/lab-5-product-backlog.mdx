## What You'll Learn

- Structure a product backlog from a PRD
- Prioritise features using value vs effort
- Estimate complexity and dependencies
- Create a roadmap for implementation

## Key Concepts

| Concept             | Description                                       |
| ------------------- | ------------------------------------------------- |
| **Product Backlog** | Prioritised list of features and tasks to build   |
| **Prioritisation**  | Ordering items by value, effort, and dependencies |
| **Story Points**    | Relative estimates of complexity                  |
| **Roadmap**         | Timeline view of planned features                 |

## What You'll Create

- A prioritised backlog from your PRD
- Feature cards with estimates and dependencies
- A rough roadmap for implementation

## Real-World Applications

- Planning development sprints
- Communicating product plans to stakeholders
- Making trade-off decisions about what to build
