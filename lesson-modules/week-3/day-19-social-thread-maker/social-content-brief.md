## Content Strategy

### Theme: "What You Can Build in 1 Hour with Claude Code"

Each day showcases a different type of product/project someone built from scratch in 60 minutes or less. Real examples from our community, with step-by-step breakdowns.

**Why this works:**

- Concrete, inspiring examples (not theory)
- Challenges "you need months to build something" belief
- Shows breadth of what's possible
- Invites participation ("I could do that!")

### Thread Structure

Each thread follows the same format for consistency:

1. **Hook tweet** (strong opening, scrollstopper)
2. **The challenge** (what was built)
3. **The starting point** (skill level, tools known)
4. **The process** (3-5 key steps)
5. **The result** (screenshot/demo)
6. **The lesson** (key insight)
7. **Call to action** (try it yourself + link)

**Total length:** 8-10 tweets per thread

## Thread Writing Guidelines

### Voice & Tone

- **Conversational:** Write like you're explaining to a friend, not giving a formal presentation
- **Specific:** Use concrete details, not vague claims ("60 minutes" not "quickly")
- **Honest:** Include struggles and failures, not just wins
- **Encouraging:** Make readers think "I could do that" not "They're so smart"
- **British English:** Honour, optimise, etc.

### Hook Tweet Best Practices

Strong hooks use:

- **Specificity:** "Built a Slack bot in 60 min" > "Built something cool"
- **Transformation:** "From zero Python to automated reports"
- **Numbers:** "Saves me 5 hours every week"
- **Relatability:** "You know that annoying task...?"

**Examples of strong hooks:**

- "I spent 5 hours/week making Excel reports. Then I built a Python script in 60 minutes that does it automatically. Here's how (no coding background required):"
- "This Chrome extension saved my content research workflow. Built it in one hour with zero extension experience. Thread ðŸ§µ"
- "A product manager built a productivity dashboard. A consultant automated client reports. Both in ~1 hour. Both with no prior coding. Here's what they learned:"

### Tweet Structure

- **Length:** 200-280 characters ideal (leaves room for retweets)
- **Formatting:** Use line breaks for readability
- **Emphasis:** Use CAPS sparingly, bold when available
- **Hashtags:** Max 2 per thread, only on final tweet

### Visual Elements

Every thread needs:

- Screenshots (at least 3 per thread)
- Before/after comparisons when possible
- Code snippets (short, well-formatted)
- Optional: Video clip or GIF showing it working

### Call to Action

**Final tweet should include:**

1. Encouraging message ("You can build this too")
2. Link to resource (beta signup, template, guide)
3. Invitation to share ("What will you build?")
4. Relevant hashtags (#BuildInPublic #IndieHackers #ClaudeCode)
