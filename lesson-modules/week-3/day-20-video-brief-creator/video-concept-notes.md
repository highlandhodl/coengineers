# Video Concept Notes: CoEngine Product Demo Series

Raw ideas, references, and creative direction for YouTube video series showcasing Claude Command Studio.

## Overall Series Vision

**Series Name:** "Build It in an Afternoon"
**Format:** Tutorial-style videos showing real builds from scratch
**Length:** 15-20 minutes per video
**Frequency:** Weekly uploads (Wednesdays at 2pm GMT)
**Platform:** YouTube (primary), with clips for Twitter/LinkedIn

**Goal:** Demonstrate that non-developers can build real, useful products using Claude Command Studio in a few hours.

## Target Audience

**Primary:**

- Indie makers and entrepreneurs (want to build but lack coding skills)
- Product managers (want to prototype quickly)
- Designers (want to bring ideas to life without developers)
- Small business owners (want custom tools, can't afford agencies)

**Secondary:**

- Developers curious about AI-assisted development
- Students learning to code
- Technical writers and educators

**Psychographics:**

- Overwhelmed by traditional coding tutorials
- Have specific problems they want to solve
- Time-constrained (need fast results)
- Prefer learning by doing over theory
- Want to ship things, not just learn

## Video 1 Concept: "Build a Habit Tracker in 20 Minutes"

### The Premise

Follow along as we build a custom habit tracker from absolute scratch. No coding experience required - just you, Claude Command Studio, and a problem to solve.

**Why this video:**

- Relatable problem (everyone wants to build habits)
- Simple enough to complete in video
- Shows both design and functionality
- Immediate value (viewers can use it today)
- Teaches core concepts without overwhelming

### Story Arc

**Act 1: The Problem (0:00-2:00)**

- Cold open: Show the final product in use (30 seconds)
- Introduce the problem: "Generic habit trackers don't fit how I work"
- What we're building: Custom tracker with exactly the features we need
- Set expectations: "In 20 minutes, you'll have a working version"

**Act 2: Planning (2:00-5:00)**

- Open Claude Command Studio
- Describe what we want in plain English
- Claude helps us break it down into steps
- Create visual workflow in the builder
- Show how Claude suggests the structure

**Act 3: Building (5:00-15:00)**

- Create the interface (drag-and-drop form builder)
- Set up data storage (Supabase integration)
- Add tracking logic (mark habits complete)
- Create visualization (streak counter, calendar view)
- Test with sample data

**Act 4: Shipping (15:00-18:00)**

- Deploy to Vercel (one-click)
- Test on mobile (responsive design)
- Make final tweaks based on testing
- Celebrate first completion

**Act 5: Next Steps (18:00-20:00)**

- Ideas for extensions (reminders, analytics, sharing)
- How to customize for your needs
- What you learned that applies to other projects
- Call to action (try building your own)

### Visual Style

**On-Screen Elements:**

- Clean screen recording (1080p minimum)
- Subtle zoom into important areas
- Highlight cursor for key actions
- Picture-in-picture: Host in corner during explanations
- Graphics overlays for key concepts

**Editing Style:**

- Fast-paced but not rushed
- Cut out dead time, keep energy up
- Background music: Subtle, upbeat, electronic
- Sound effects: Minimal (success chimes, transitions)
- Text overlays: Key points and timestamps

**Host Presence:**

- On camera for intro and outro
- Picture-in-picture during build (top-right corner)
- Full screen during key explanations
- Enthusiastic but authentic (not over-performed)

### Script Structure

**Intro (On Camera):**
"Hey, I'm Sarah from CoEngine. Today we're building a custom habit tracker from scratch in about 20 minutes. No coding experience required - I'll explain everything as we go.

By the end of this video, you'll have a working habit tracker deployed to the web that you can use on your phone or computer. More importantly, you'll understand the process of building custom tools, which you can apply to any problem you want to solve.

Let's dive in."

**Transitions Between Sections:**

- "Alright, now that we know what we're building..."
- "This is where it gets interesting..."
- "Almost done - just need to..."
- "And just like that, we have a working app!"

**Teaching Moments:**

- Pause to explain why we're doing something
- "You might be wondering why..." followed by clear explanation
- Point out patterns: "This is something you'll do in every project..."
- Acknowledge complexity, then simplify: "This looks complicated, but it's really just..."

**Outro (On Camera):**
"So there you have it - a custom habit tracker built in 20 minutes. You can customize this to track whatever matters to you, add reminders, build analytics, whatever you need.

The process we followed - describe the problem, break it down, build step by step - works for any project. Next week, we're building a client onboarding system.

If you build your own version, share it on Twitter and tag us @CoEngine. I'd love to see what you create.

Links to the template and Claude Command Studio are in the description. Thanks for watching!"

### B-Roll & Supplementary Footage

**Insert clips of:**

- Final product in use (on phone and desktop)
- Quick preview of more complex examples
- Other people's habit trackers (inspiration)
- Deployment success screen
- Celebration moment (subtle, authentic)

### Lower Third Graphics

**Information to display:**

- Host name and title: "Sarah Chen, Head of Product @ CoEngine"
- Key stats: "No coding required" / "20 minute build time"
- Tool names when introduced: "Claude Command Studio" / "Supabase" / "Vercel"
- Website/social: "@CoEngine" / "coengine.com"

### Mistakes to Avoid

- Don't go too fast (pause to let concepts sink in)
- Don't skip error handling (show what to do when things go wrong)
- Don't assume knowledge (explain acronyms and concepts)
- Don't make it look too easy (acknowledge learning curve)
- Don't be sales-y (focus on teaching, not promoting)

## Production Requirements

### Equipment Needed

- **Video:** 1080p screen recording (4K for future-proofing)
- **Audio:** Studio-quality microphone (crisp, no background noise)
- **Camera:** For host segments (DSLR or high-quality webcam)
- **Lighting:** Soft, even lighting for host segments

### Software Stack

- **Recording:** OBS Studio or ScreenFlow
- **Editing:** Final Cut Pro or Premiere Pro
- **Graphics:** After Effects for animations
- **Audio:** Audition for post-processing

- [ ] Script written and approved
- [ ] Build tested end-to-end (no surprises during recording)
- [ ] Screen recording setup tested

- [ ] Music licensed and downloaded

- [ ] Rough cut completed
- [ ] Audio cleaned and leveled
- [ ] Graphics and lower thirds added
- [ ] Colour correction applied
- [ ] Transitions polished
- [ ] Captions generated (for accessibility)
- [ ] Thumbnail designed
- [ ] Description and tags optimized for SEO
